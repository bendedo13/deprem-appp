═══════════════════════════════════════════════════════════════
🚀 VPS'TE ÇALIŞTIRMANIZ GEREKEN KOMUTLAR
═══════════════════════════════════════════════════════════════

1️⃣ VPS'e Bağlan:
───────────────────────────────────────────────────────────────
ssh root@your-vps-ip


2️⃣ Proje Dizinine Git ve Güncellemeleri Çek:
───────────────────────────────────────────────────────────────
cd /opt/deprem-appp
git pull origin main


3️⃣ Deploy Script'i Çalıştır:
───────────────────────────────────────────────────────────────
cd /opt/deprem-appp/deploy
bash deploy.sh

Bu script otomatik olarak:
✅ Docker container'ları durdurur
✅ Yeni image'ları build eder
✅ Container'ları başlatır
✅ Database migration çalıştırır
✅ Health check yapar


4️⃣ Logları İzle:
───────────────────────────────────────────────────────────────
docker logs -f deprem_backend
docker logs -f deprem_celery


5️⃣ Servisleri Kontrol Et:
───────────────────────────────────────────────────────────────
docker compose -f /opt/deprem-appp/deploy/docker-compose.prod.yml ps


═══════════════════════════════════════════════════════════════
⚡ TEK SATIR DEPLOY KOMUTU
═══════════════════════════════════════════════════════════════

cd /opt/deprem-appp && git pull origin main && cd deploy && bash deploy.sh


═══════════════════════════════════════════════════════════════
⚠️ YANLIŞ KOMUTLAR - KULLANMAYIN!
═══════════════════════════════════════════════════════════════

❌ YANLIŞ (tehlikeli - main'i feature branch ile ezer):
git reset --hard origin/claude/...
git push origin main --force

✅ DOĞRU (sadece güncellemeleri çeker):
git pull origin main


═══════════════════════════════════════════════════════════════
📝 YARDIMCI KOMUTLAR
═══════════════════════════════════════════════════════════════

# Container'ları yeniden başlat
docker compose -f /opt/deprem-appp/deploy/docker-compose.prod.yml restart

# Container'ları durdur
docker compose -f /opt/deprem-appp/deploy/docker-compose.prod.yml down

# Tüm logları izle
docker compose -f /opt/deprem-appp/deploy/docker-compose.prod.yml logs -f

# Health check
curl http://localhost:8001/api/v1/health
